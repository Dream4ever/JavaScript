# 第七课 - 分享职场心得

## 开场

好几周都没有给大家上课了。首先恭喜那些找到工作的同学，老师觉得非常开心，因为能够帮到大家。这些东西其实大家只需要坚持努力一年，薪资水平一定可以翻倍，一年翻倍是很正常的。不管第一份工作是什么内容，都要踏踏实实地去做，每天都要坚持学习，一年之后薪水 double 没有任何问题，甚至可以翻三倍。老师最开始工作的时候，第一个月的薪水非常低，但是三个月转正之后就直接 double 了，一年之后则是刚入职时的四倍。有人可能说老师最开始的工资很低，但是重点应该关注在“增长速率”上。所以大家要靠自己的成长速度去跑赢什么物价的上涨、卫生纸的涨价，那些事情都是浮云。自己强大了，还怕别的么？不用关注这些事情，只需要关注自己明年的薪资水平要达到什么层次，自己应该向着什么方向努力，就可以了。

老师最近在为新东家招聘，后面也会聊一下从新大解散到进入新东家的心路历程，会和大家分享一下自己是如何思考的，可能会对大家的职业发展有一定的参考价值。

## 如何转型

老师在去年 11 月份和笑来老师因为项目合作的原因，把新大的开发团队解散了。但是出于为团队负责的原因，徐老师和张越老师运用自己的人脉，帮大家迅速找到了下一份工作，不至于在过年的时候还没有工作。好在这个团队已经磨合了两年左右的时间，团队里的每一位同学都很出色。

就拿新大的前端 leader 来说，干过厨子，学过武术，打过工，报过培训班，学得也不咋地。来徐老师这里直接说工资随便开，我就是要锻炼自己。老师也是第一次遇到这种应聘者，也是蛮头大，哈哈。最后还是让那位同学提了个要求，虽然低，但是符合他当时的情况。让老师欣慰的是，这位同学真的是只用半年的时间，就从一个门外汉（非科班出身，报班学得也不咋地）成为了中级工程师。他在前三个月是游离状态，可能是刚入职互联网公司时比较谨慎，不知道该怎么做，所以在调整状态。之后又用了三个月时间迅速提升自己的能力，把自己从门外汉提升成了中级工程师的水平。即使在前三个月的游离状态，他也是花了相当大的功夫来打基础的。所以老师敢说大家可以靠三到六个月就可以成功转型（拿 JS 这门语言来说）。这位同学并不比大家聪明，关键的是新大这么一个能够让大家迅速成长的团队。这样的运气并不是每个人都能有的，而这位同学恰恰遇到了这样一个环境。

所以老师这里要给大家一个建议，去了一家公司之后，不要去否定别人，而是应该不断地否定自己，想想自己为什么提高得这么慢，而不是去想公司的项目怎么这么垃圾，这跟自己没关系。只要公司能给薪水，就踏踏实实地去完成工作上的任务，并且充分利用一切机会提升自己。所以工作不满两年的人要清楚自己的目标，不是要帮公司上市什么的大目标，而是老老实实地完成 leader 给自己布置的任务，另外一个很大的目标就是不断不断地学习，这种学习的压力应当比高三高考的压力还要大。大学不就是锻炼自学能力的么，同时也锻炼抗压能力。一定要注意这一点。公司对员工的期望不会太高，只要完成任务就可以了。毕竟一家成长型的公司也是会有不同层次的员工的。

说到公司员工的层次，老师现在也在想，公司的专家要多少人，中高级工程师要多少人，初级人员要多少人。在一个健康的技术团队中，初级人员一定是不多的，中高级工程师是占比很多的，这样一个团队才能够带领公司飞速发展。其实公司都不愿意去找初级工程师，因为他们效率低，但是为什么还会去找呢？第一是想培养出自己的初级工程师，因为直接找现成的高级工程师成本很高，不管什么样的公司，都想培养出自己的人才，所以更多地是希望初级工程师快速地融入团队，快速地提升能力，公司并不指望初级工程师带来很多的贡献，只是希望初级工程师们每天都在成长，这才是关键，这样过三四个月能开始贡献出自己的力量就可以了；第二点也是因为找不到满意的高级工程师，所以就先拿初级工程师顶着，但是这种情况下效果往往并不好，妥协的代价就是，过不了多久整个系统的代码就要重写一遍。一个企业招初级工程师就是这两点原因，是很现实的普遍现象。所以大家并不用担心找不到工作，而是在进入公司之后要珍惜机会，不要去否定别人，而是要专注于自己的两个目标，一个是认认真真地、仔仔细细地完成分配给自己的工作，第二个就是不断学习、不断学习、不断学习。说来说去就是那些东西，要天天练、天天学，要保持这种不断成长的状态，大牛都是这样成长起来的。

再回到新大团队的解散，团队解散之后，各位同事有的去了京东，有的去了美团，有的去了滴滴。去了滴滴的同学很有意思，他是徐老师挖过来的，这位同学很有意思，也很有魄力。他原来在国企，做的是 Java 的服务端。老师当时在 Node.js 社区发了招聘需求，这位同学给徐老师发了简历。周末在咖啡馆约了一次之后，老师当即拍板决定要他。这位同学当时并不符合公司的技术栈，公司用的是 Node.js，但这位同学用的是 Java，也就是说他来了之后不能立刻上手，还需要先学习一段时间。那么老师看中的是这位同学的哪一点呢？他虽然在稳定的国企，却感觉度日如年，每天做的事情也没有压力，觉得自己学不到东西，于是想离开国企去看看真正的互联网。而且他来了新大之后，薪水要的并不比国企高很多，大概也就提升了 20%。他刚来虽然什么都不会，但是快速地提升自己，两三个月之后就成为了团队中一名核心的维护人员。现在他去了滴滴，这家公司怎么也是中国排行前五的互联网公司，说明这位同学也是很厉害的。

如果把新大作为跳板的话，这些同学都跳的非常成功。老师很欣慰的就是大家在新大一起成长、一起做事，大家能够快速地提升自己的能力和意识，这就是团队的价值。事情的成败先不论，但是大家每个人都获得了成长，这是非常有意义的一件事。前面说到的那位前端 leader 去了京东，现在已经是高级工程师了。

这里老师要说个题外话，之前韩巍同学和徐老师在微信上单独聊了聊，老师觉得在微信上不方便深入展开，于是就打电话交流了一番。当时韩同学就问 Ruby 在国内的前景究竟如何？要不要学？老师当时的观点是：以前就强调过，在中国的互联网环境里，Ruby 不是一个架构师的首选语言。为什么 XDite 会来北京宣传 Ruby 呢？因为 XD 老师只会 Ruby，而且 Ruby 是台湾地区的架构师的首选语言之一，特别是 ROR。为什么 Ruby 在台湾这么受欢迎，反而在大陆不是这样呢？这个跟环境有关系，因为台湾受日本文化的影响比较深，而 Ruby 的作者也是日本人，所以 Ruby 在台湾的推广就比较好，大陆这边就不一样了，所以 XD 老师在大陆的 Ruby 教学并不太受欢迎。毕竟大家参加培训就是为了找工作，如果报班培训之后不能很好地找工作，那么这个培训班自然就难以为继。

## 徐老师的心路历程

徐老师自己其实也在不断地抉择。第二期课结束之后，徐老师说自己不太愿意去做培训，因为老师觉得自己还需要积累，如果一直做培训而不是在一线工作的话，过不了三年就很难掌握到互联网行业技术动态了。虽然这三年做培训的话可能还可以赚钱，但是三年之后就不好说了。虽然为了保持技术敏锐度可以和自己圈子里的人交流，但是毕竟还是比自己去一线历练的效果要差很多，对各方面的理解都不如后者深刻。后来徐老师和身边的人不断沟通之后又觉得自己还是很想去做培训，也慢慢摸索出了自己的方法，到了最后又不想做了。年前的时候徐老师差点儿又回阿里了，各种事情基本上都要谈妥了，但是在年后徐老师又把阿里给拒了。阿里云是目前中国做云服务最好的一家公司，每年的收益、财报都很漂亮，那么徐老师为什么放弃这个机会呢？徐老师就会想，四年之后呢，六年之后呢？精力跟不上了该怎么办呢？徐老师想明白了一件事情，我是不是要一直在技术一线奉献一辈子？除非到了一个非常高的位置（总监以上），不然年纪大了之后就会处在一个很尴尬的境地。处在徐老师的这个年龄，就需要着手去规划自己未来的职业路线了。群里的李朋同学年前和徐老师交流过一次，李朋同学和徐老师的情况有点类似，也处在一个比较尴尬的状态。对于徐老师来说，并没有处在一个比较尴尬的状态，而是在主动为未来的方法做构思，这件事情一定要趁早做。一个人在二十多岁的时候可以被动，可以跳槽。但是做了很多年之后就不能被动地去做这些事情，而是应该更加主动地去思考自己的职业发展。所以老师就是因为这个原因拒绝了阿里的 offer，而是选择去做汽车。徐老师个人是想踩中国内的一个热点，为什么会想踩这个热点呢？人工智能、AI、大数据、智能驾驶，这都成为了国家的主打方向了，就需要有人为之奋斗。徐老师也想了很久，就加入了这样的一家公司，带着兄弟们去做这件事。很有意思的是，第一个试点的地方就在青岛（啊哈哈哈），后面可能会经常去青岛（必须要请你吃饭，哈哈）。

关于编程培训这件事情，徐老师永远不会放弃，免费的也没关系，因为徐老师并没有想在这件事情上去赚钱，只是想把这件事情做好。徐老师现在想把这件事情 share 给大家，以目前这个社群为基础，不断地吸收新鲜血液，先不管这个社群最后会成为什么样子，我们就把它当做让我们实现个人成长的机会，每个人的能力点不一样，大家都是各不相同的，这样一群人凑在一起，肯定是可以把这件事情做成的。所以徐老师非常期待大家能够迅速成长，组成一个团队去做这件事情。在十年之内，编程肯定会主导国家发展的。今天最重要的一点，就是勾引大家，一起努力。反正都坚持这么久了，那么肯定还能坚持更久。大家刚开始也不要想着去赚钱，而是把它当成提升自己的时候顺便做的一件事情。它不需要花费我们很多的精力，反正每天都要去学习。假如有一天这个团队真的能够做成一个社区化的编程培训的东西的话，那么徐老师也是会很欣慰的。

所以徐老师未来的职业发展是两点：先做好自己的工作，借助着这个大力发展高科技行业的浪潮，做出一番事业；在财富自由之后，专心地去做培训，帮大家进入这个行业。第二点就是时刻关注自己的理想，时刻去积累，老师的理想也可能会变成大家的理想。这两点会持续在老师的个人发展过程中，这就是老师要给大家分享的。

之前老师想找几个实习生到自己身边工作，但考虑了很久之后，暂时不会这么做了。因为业务上有压力，所以希望招的人进来之后能够迅速上手去完成所分配的任务。后期业务跑顺之后可能会想着培养一些人才，如果哪天老师真的说了这件事，大家就尽管挑战一下自己吧。老师目前一直在调整结构，业务结构，人员结构，思考业务未来的发展方向。所以大家就不断努力，未来要是真能在一个团队中共同成长，那该多开心！

有很多同学前面在面试的时候就说老师讲 Vue 时的知识很管用，而且也是因为 Vue 是中国人做的，所以大家都会对这个框架有些偏向。这就是有意思的地方，台湾的技术圈比较偏向 Ruby，大陆比较偏向 Vue，等等。其实团队的架构师、核心骨干在决定技术方向的时候，里面是有很多人情味的东西在里面的。比如喜欢在变量命名上把女朋友们打印出来，大家可以充分利用自己的聪明才智去把人情味的一些东西融入进去。

做架构的时候尤其如此。架构没有一成不变的，每家公司都有自己特色的架构。京东和淘宝都是做电商，业务模式也差不多，但是架构肯定是不一样的。即使京东把淘宝的架构师挖过去，架构师肯定也不会照搬阿里的架构，也是一定会根据京东的现状，比如技术团队的水平，来决定公司的架构。这一点就是最好玩的，它是可以变化的。老师在做现在的这件事情的时候也会去想，之前在新大做直播时所用的架构，优点在哪里，但肯定不会把这个架构照搬过来。做架构的乐趣就在这里，人的因素，环境的因素，业务的因素，和打仗一样，有各种计谋策略，如何运用都要看实际情况。老师觉得做架构就是一种乐趣所在，把各种因素通盘考虑之后，确定最适合当前业务的架构，这就是老师工作的价值。挑战很大，但是也很有意思。女工程师有合适的还是会招，但关键的还是要看能力，一定要能够完成公司的业务才行，所以其实男生还是女生并不重要。大家在工作的时候还是比较忌讳办公室恋情，会很影响工作的正常开展，特别是在技术讨论的时候，两个人的业务有交叉的时候。工程师毕竟还是比较容易情绪化的一个工种，徐老师以前也是，“你个死脑筋，多传个参数会死嘛？”搞技术的总会觉得自己一定就是对的，不能够妥善处理工作中遇到的问题的话，就会影响工作。

徐老师的第一家公司就是，你们可以谈，但是其中一个人必须离职，不管是什么级别。或者就两个人都走，当然了，其中一个级别很高的话，可以跨部门调转。

说到徐老师最近两个月的选择，有时候选择多了还很烦，没有选择反而很舒服。抉择来抉择去，不管是阿里云还是现在的公司，两边都是从十一月底就开始接触了。阿里云毕竟熟悉，很多方面谈得还是比较好的，当时差点儿就去飞猪了，但是会到处出差，让人很烦，所以把飞猪拒掉了。选择多了之后，拒绝任何一方都很不好受。未来大家也会遇到这种问题，在招聘员工的时候应该选谁，是一件很头疼的事情。

选择互联网公司的三个准则是永远不会变的：

1. 要了解自己的直接上司（目标必须一致）。先看看那些主导大公司的人，虽然在创业过程中会常常失败，但他们的想法都很坚定，不会被困难和诱惑所打败。如果总是受外界干扰，方向变来变去，那只是打游击战，不去实现个人价值，就很难成大事。马云和刘强东都是如此，都做电商，也有竞争，但其实并不冲突。回归正题，选择工作的时候，倒不用太关注 CEO，关键的是要主动去了解未来的 leader 是谁，在招低级工程师的时候，leader 一般都是要出面的。所以面试其实是一个双向选择的过程，leader 要考察面试者，面试者也要考察 leader。在面试过程中，如果面试到最后，问应聘者有什么问题，没经验的人就常常会问“有五险一金么？”“公司加班么？”，这你让面试官怎么回答？需要加班的时候必须得加班，这还用问嘛？所有的这些问题都是和切身利益相关的，但是徐老师要告诉大家，在成为高级工程师之前，都不应当太关注这些东西。最应该关注的，是公司能不能给予员工更多的成长。比如说可以问“现在公司前端的结构是什么样的？有多少高级工程师？像我这样水平不高的，公司会用什么样的方式培养、引导我？公司会如何让我迅速提升？”不要问那些无聊的问题，如果打车不报销，那就不去了？一定要问出真正有价值、能够帮助到自己的问题。如果公司对于徐老师所建议的问题给不出让人满意的回复，那么说明这家公司不太值得去。一家优秀的公司，虽然工资可能少一些，但是对于员工的提升是远远超出工资的价值的。要让公司知道，应聘的这位员工非常关注自己的成长。对于初级工程师，面试时有一个非常重要的点要不断地去打磨：如何用各种方式方法，在面试的过程中透露出这么一种意向，就是自己是一个非常努力、非常刻苦、愿意吃苦、愿意付出很多代价去提升的人。老师去年面试过一个 Vue 前端，这个应聘者问的所有问题都是和切身利益相关的，而徐老师问了很多深入的技术问题之后，这个应聘者都回答不上来。比如 Promise 的 then 函数返回的到底是什么类型，应聘者不仅没有答对（答案是 Promise 类型），而且还和徐老师死磕，这就很尴尬了。在面试的过程中千万不要制造这种尴尬的情况，没有这个必要。

2. 要了解自己即将加入的团队。可以问团队中各个层级员工的比例结构，如果公司中有几个大牛，就会很自豪地说出来，这也是了解团队信息的一种方式，面试官所透露出的各种细节都可以作为判断依据。反过来说，面试官也会觉得这个应聘者很有想法。也可以问问团队中大家的性格都是什么样的，这样能够大概知道团队中的成员是否好配合。

3. 真正要去做的事情。不同的层级会有不同的看法，高层次的会去想这件事情能不能成，能不能融资，能不能做到像滴滴、摩拜那么大。底层的要去看团队的技术栈是否和自己的方向相同，是否符合自己的发展路线。假如自己的 Vue 很牛了，但是要加入的团队却只愿意用 jQuery，那么说明这样的团队不太适合自己。

工作不到两年的同学都不要太关注公司能给多少薪水，最应该关注的是公司、老板能给自己带来多大的成长。不管是创业还是小公司、大公司等等，都要关注老师上面说的这三点。上面三点的优先级不会变，只不过会根据自己的层次不同，思考的角度会有所变化。举个例子，如果前两点都完全 OK，而公司的技术栈是 Ruby，自己也愿意尝试的话，其实这家公司也完全 OK 啦。在老师看来，工作几年之后还只会掌握一种语言的人早晚都会被淘汰掉。一个合格的开发者至少应当精通两门语言。

至于是否要补一个软件开发方向的研究生的学历，谁也给不出准确的回答。这种东西有了当然是好，但是没有也不一定会差。如果能够遇到一个很欣赏自己的人，那么没有这个也完全没关系啦。有些学历低的人很多年都在小公司跳来跳去，成为大牛了之后这个学历的影响可能就不那么重要了。

今天没有上课，和大家分享一下是很有价值的。这个编程培训的事情老师觉得是非常有意义的，希望大家都能够努力成长，大家一起干一票大的！

Vue 是非常重要的，大家要不断地去学习。明天晚上老师会给大家讲一下 Vue-router 的使用技巧，这个插件能够解决很多问题。老师会给大家解释一下，切换路由的时候，切换的是组件还是页面。之前老师解释过概念，明天老师会通过实际示例让大家真正理解这个概念。

为什么会招 Java 后端？因为项目现有的后端都是 Java，团队会的也都是 Java。现有的人员结构不可能改变，所以只能不断地去巩固它。另外 Java 做后端完全没问题，而且它的生态比 Node.js 更好。老师所说的这个生态，有两种意思。语言意义上的生态，Node.js 是很出色的。架构/安全方面的生态，Java 则是比较优秀的。拿阿里来说，80% 的服务端用的都是 Java。

精通两门语言并不难。首先，要先精通一门语言，之后再学习其它语言的时候，就有自己的方式、方法了，因为知道什么是重要的，什么是不重要的。拿 JS 来说，要精通这门语言，需要掌握哪些知识呢？老师在前面的课程中都讲到了，大家只是需要时间积累而已。

Python 能够在机器语言中这么流行，是因为它的库很丰富。精通 JS 和 Python 其实区别不大，相当于同一种语言。两种语言是指不同类型的语言，一门是动态语言（JS/Python），另一门是面向对象语言。

Golang 这门语言很好，也不难。只不过有一些概念是动态语言所不具有的，有些特性更接近 C++。当初在搭建用户系统的时候，同样的一批人，用两种语言写的代码，Golang 的性能提升了接近 30%（概数）。比如以前的新大 APP 直播经常进不去，徐老师进来之后就好多了。

归根结底，每种语言都有它的价值所在，就要看自己能不能把它写得更好。

所以在以后的发展过程中，一定要紧跟大公司的技术栈走向，比如 Facebook、Google、Amazon、Twitter，国内的话就是 BAT。多关注它们的技术走向，所用的编程语言。

测试性能的手段：其实很简单，比如一秒钟发一千个 HTTP 请求，看响应时间。当然也有专门的工具，比如 jmate，这样的工具是很多的。所以有些同学想从功能性测试转到性能性测试，根本不用学编程语言，只要学会了一种工具，再学会 HTTP 的基础知识（Query、Get、Post、Body、Cookie、Head），就已经可以完成性能测试的工作了。比如测试单点抗压能力，给登录接口一秒钟发一万次请求。也可以测试事务，一个用户从打开 APP 到展现 APP 内容的过程中调了多少接口，把这些接口列出来，然后在测试工具中去模拟用户操作，比如测试一千个用户的并发，看看服务器处理完这些事务的响应时间，也许还能压出逻辑问题。测试这件工作其实很简单，只是大家不会用工具而已。

把 jmate 这样的工具研究透了之后，再掌握两点：单点测试，事务测试，就肯定能够找到工作。

就徐老师个人而言，面试的时候非常注重基础，这是最关键的。把编程语言的基础打牢，掌握好吃饭的看家本领，后面就海阔凭鱼跃，天高任鸟飞了，哈哈。

用 vue-cli 创建出来的项目，会发现项目中没看到 webpack，为什么？因为它有自己的打包方式，但是并不够全面。所以在公司内部中做项目的时候，并不会直接用 vue-cli 直接创建，而是会用 Vue 官方文档中列出的 Webpack 模板。老师建议大家用它去创建一个项目，和 vue-cli 进行对比，然后就会发现，这个 Webpack 模板才是更强大的一个东西。
